## 修复 theme-switcher.tsx 报错（第二轮）
- **问题**：三个错误同时存在：
  1. `<Script dangerouslySetInnerHTML>` 仍然触发 script tag 警告
  2. `localStorage.getItem is not a function` — SSR 阶段 localStorage 不可用
  3. `updateDOM is not a function` — 内联脚本未正确执行
- **最终修复方案**：
  1. 创建 `public/no-fouc.js` 静态脚本文件（外置 JS 文件，彻底绕开 React 渲染管道）
  2. `layout.tsx` 中用 `<Script src="/no-fouc.js" strategy="beforeInteractive" />` 加载
  3. `theme-switcher.tsx` 重写：
     - 删除 `NoFOUCScript`（已迁移到静态文件）
     - `useState` 初始化时用 `"system"` 作为默认值，localStorage 读取移到 `useEffect`（仅客户端执行）
     - 用 `safeGetItem`/`safeSetItem` 包装 localStorage 操作，防止 SSR/隐私模式报错
     - `updateDOM` 取自 `window.updateDOM ?? (() => {})`，加可选链调用
- **关键经验**：
  - Next.js App Router 中任何形式的内联 `<script>` 标签（直接 HTML、dangerouslySetInnerHTML、next/script 的 dangerouslySetInnerHTML）都会触发警告
  - 正确做法：创建 `public/` 下的静态 `.js` 文件，用 `next/script` 的 `src` 属性加载
  - 任何 localStorage/indexedDB/Document API 在 `useState` 初始化阶段都不可用，必须放在 `useEffect` 里
