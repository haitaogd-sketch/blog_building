# 2026-04-05 工作日志

## Vercel 部署问题排查

### 问题描述
Next.js 项目通过 GitHub 推送到 Vercel 部署后，显示 404 错误。

### 根本原因
1. 缺少 `next.config.ts` 配置文件
2. Vercel Root Directory 设置为 `/`（仓库根目录），但项目实际在 `blog-starter-app/` 子目录下

### 解决方案
- 创建 `next.config.ts` 配置文件
- 将 Vercel Settings → General → Root Directory 从 `/` 改为 `blog-starter-app`

### 经验总结
- Vercel 的 Framework Preset 设置可能存在缓存，需要通过修改 Root Directory 或触发新部署来刷新
- 项目在子目录时，Vercel 需要正确设置 Root Directory 才能找到 package.json
